# Monster Workshop 開發待辦清單

更新日期：2026-07-06

## 執行原則

- 依階段順序推進；標記為「無依賴」的內容可以穿插開發。
- 新增或修改狀態結構時，必須同步更新 `saveVersion`、遷移函式與驗收清單。
- 每完成一項功能，需執行 [CHECKLIST.md](CHECKLIST.md) 中受影響的測試。
- 階段三的競技場是後續屬性、遠征事件與 Boss 系統的共同驗證場。
- 連線與 DLC 必須等核心資料格式穩定後再開始。

## 階段依賴

```text
階段一：半成品補完
  ├─ 階段二：獨立內容
  ├─ 階段三：怪物構築驗證 V1
  │    └─ 階段五：屬性系統 V2
  │         └─ 階段六：遠征隊事件 V3
  │              ├─ 階段七：Boss／守護者 V4
  │              └─ 階段八：遠征遺物 V5
  └─ 階段四：個性／心情關係群組

核心系統穩定
  └─ 階段九：結構性擴張
       └─ 階段十：連線功能
            └─ 階段十一：DLC 內容
```
